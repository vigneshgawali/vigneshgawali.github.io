Projects

  • Hermes - Analysis of New York Taxi Data

    This project is aimed towards generating taxi trip trends related to 45 different locations throughout New York City. The data input to the system is a New York City Yellow Taxi public data source from 2010 to 2015, used to derive statistics related to each location, for e.g. Total Number of Trips, Trips during different times of day, Number of trips during weekends and holidays, Total tip amount, Trip distance etc. This derived information is then used to generate visual trends and can be compared with multiple locations.

    The above generated visual representations give detailed insights related to trips. For e.g. We found that the average tip/mile (tip per mile) amount has risen from $0.35 in 2010 to 0.80 in 2015. Also, people travelling to movie theaters pay more tip on an average as compared to people travelling to other locations.

    • Role Played: Development and Visualization
    • Team Size: 3
    • Skills Used: PySpark, Tableau

  • Packet Loss Detection and Recovery

    The traditional TCP protocol remains inefficient in the process of retransmission of packets lost over the network. This project aims at implementing the TCP Constant Packet Reordering technique, which uses timers instead of Duplicate Acknowledgements, resulting in more efficiency and less congestion as compared to traditional TCP.

    In conjunction to CPR mechanism, we also implemented Random Early Detection queueing technique that calculates the probability of the packet being dropped for every packet being added to the sender queue. The implementation of these techniques resulted in higher efficiency of retransmission of the lost packets, and double the throughput as compared to traditional TCP protocol.

    • Role Played: Development and Simulation
    • Team Size: 3
    • Skills Used: Network Simulator, TCL Script, AWK Script
